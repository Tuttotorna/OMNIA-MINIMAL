# Release Checklist

This checklist defines the release procedure for OMNIA-MINIMAL.

---

## Pre-release checks

Before creating a GitHub release, verify:

    python examples/run_minimal_demo.py
    pytest -q

Expected test result:

    4 passed

Also verify that GitHub Actions is green.

---

## Required release files

The repository should contain:

    README.md
    START_HERE.md
    CHANGELOG.md
    RELEASE_NOTES.md
    CITATION.cff
    .zenodo.json
    docs/MINIMAL_PAPER.md
    docs/REPRODUCIBILITY.md
    docs/FAILURE_CONDITIONS.md
    docs/WHAT_TO_READ_NEXT.md
    docs/ECOSYSTEM_READING_PATH.md
    docs/CLICKABLE_ECOSYSTEM_INDEX.md

---

## Tag

Current release tag:

    v0.6.0

---

## GitHub release title

Use:

    v0.6.0 - Structural Stability Beyond Surface Correctness

---

## GitHub release body

Use the contents of:

    RELEASE_NOTES.md

---

## Zenodo

If Zenodo is connected to the GitHub repository:

1. Create a GitHub release.
2. Zenodo should archive the release automatically.
3. Copy the generated DOI.
4. Add the DOI to:
       CITATION.cff
       README.md
       RELEASE_NOTES.md

Do not invent a DOI manually.

Only use the DOI generated by Zenodo.

---

## Boundary reminder

Do not overclaim the release.

Correct statement:

    OMNIA-MINIMAL demonstrates that correctness and structural stability can diverge.

Incorrect statement:

    OMNIA-MINIMAL proves truth.

Boundary:

    measurement != inference != decision
